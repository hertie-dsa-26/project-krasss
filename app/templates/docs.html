Documentation

Climate & Health Explorer is a Flask-based web application for exploring relationships between weather patterns and health outcomes across U.S. counties (2013–2023), using data from NOAA, CDC, and the U.S. Census.

Architecture overview

The ML implementation is split into four classes, each responsible for a distinct part of the pipeline. No external ML libraries (e.g. scikit-learn) are used — all components are implemented from scratch using NumPy and Pandas.

ClassResponsibility
ModelK-fold splitting, train/test split, cross-validation
PreprocessingImputation, encoding, scaling (inherits from Model)
GaussianKernelRBF kernel computation and kernel matrix construction
KernelRidgeRegressionModel fitting and prediction using dual coefficients
AssessmentR² score, MSE, cross-validation scoring (inherits from Model)

Preprocessing

The Preprocessing class implements the following transformations from scratch:

MethodDescription
SimpleImputerReplaces missing values with the column mean
OneHotEncoderEncodes categorical features as binary indicator columns
StandardScalerStandardizes features: z = (x − μ) / σ
ColumnTransformerApplies transformers to column subsets and concatenates results

Gaussian (RBF) kernel

The GaussianKernel class computes similarity between data points using the RBF kernel:

K(x₁, x₂) = exp( −γ · ‖x₁ − x₂‖² )

Where γ (gamma) controls the rate of decay — a smaller gamma means similarity falls off slowly with distance, a larger gamma means it falls off quickly. The full kernel matrix K is computed pairwise over all training samples and passed to the KRR model.

Kernel Ridge Regression

Fitting

Training solves for the dual coefficients α by inverting the regularized kernel matrix:

α = (K(X, X) + λI)⁻¹ y

Where λ is the regularization parameter (L2 / ridge penalty), added along the diagonal of the kernel matrix before inversion.

Prediction

For new inputs, predictions are computed as:

f(x) = K(x, X_train)ᵀ · α

Cross-validation

The Model class implements k-fold cross-validation from scratch. It also supports a Leave-One-Out CV (LOO-CV) closed-form shortcut, which avoids retraining the model n times:

e_LOO = ( (yᵢ − (Kα)ᵢ) / (1 − Hᵢᵢ) )²

Where H = K(K + λI)⁻¹ is the hat matrix and Hᵢᵢ is its i-th diagonal element.

Parameters

ParameterDescription
alpha / λRegularization strength — higher values reduce overfitting
γ (gamma)RBF kernel width — controls how quickly similarity decays with distance
kNumber of folds for cross-validation

Assessment

The Assessment class implements two evaluation metrics:

MetricFormulaInterpretation
R²1 − (RSS / TSS)Proportion of variance explained; 1.0 is perfect
MSEmean((y_test − y_pred)²)Average squared prediction error; lower is better
