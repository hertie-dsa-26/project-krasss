Data Engineering Pipeline

PHASE 1
1

Climate Aggregation

Aggregate NOAA stations → MSA-year level. Compute absolute values (avg temp) + year-on-year changes (Δ temp). Validate station coverage per MSA; exclude MSAs below threshold.

2

Climate Zone Classification

Assign each MSA to a NOAA climate region. Use region as a control variable in the model and as a grouping layer in exploratory analysis. Enables region-level fallback for MSAs with thin data coverage.

3

Merge & Clean Master Panel

Join CDC Places + NOAA + Census on MSA identifier × year. Audit missing data; document drop rate. Check CDC Places variable consistency across all years. Group into three themes: Weather / Demographics / Health. Output: one clean MSA-year panel.

clean MSA-year panel

Statistical Analysis

PHASE 2
4

Descriptive Climate–Health Analysis

Summary stats, distribution plots, correlation matrices (climate ↔ health, climate ↔ demographics). 10-year trend lines per outcome grouped by MSA and climate zone. Both absolute and year-on-year relative climate values computed and compared. Feeds directly into /explore page.

5

Climate Gradient Comparisons

Primary: compare health outcomes across climate quintiles — hottest vs. coolest MSAs, wettest vs. driest, most vs. fewest extreme heat days. Secondary: compare across NOAA climate zones for geographic map visualisation.

validated signal

Modelling

PHASE 3
6

Predictive Model — Linear Regression (main model)

X: NOAA climate variables + Census demographic controls + climate zone classification
Y: each of the 4 CDC Places outcomes separately
Also used for 5 and 10-year forward projections with confidence intervals. Climate zone fallback for MSAs with limited data, displayed with a clear limitation warning.

7

Model Evaluation

Cross-validation across MSA-years + out-of-time test (train on earlier years, test on most recent year). Assess whether absolute or relative climate values perform better as predictors. Evaluation stats documented in /docs page for researchers.

Stretch Goals

OPTIONAL

Random Forest

Upgrade on linear regression. Same X and Y. Captures non-linear climate-health relationships. Core app runs on LR alone if not completed.

SHAP Feature Importance

Requires Random Forest. Identifies which NOAA variables drive predictions. Decomposes into climate vs. demographic contributions.

Fixed-Effects Panel Regression

MSA + year fixed effects. Framed as association analysis, not causal estimation. Limitations flagged explicitly in /docs.

Spatial Autocorrelation

Moran's I + LISA cluster maps on CDC Places outcomes. Requires geopandas. Most policy-friendly visual output in the app.

Extreme Event Studies

Use NOAA extreme temp / precipitation variables to examine whether shock years produce measurable CDC Places health spikes.
